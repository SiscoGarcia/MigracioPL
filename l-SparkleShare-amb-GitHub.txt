1.- Descarregar i instal·lar el Git

	$ sudo apt-get install git-core git-gui git-doc

		Els següents passos es poden obviar i passar directament a la creació d'un parell de claus:	
		$ cd ~/.ssh
		$ ls
		$ mkdir key_backup
		$ cp id_rsa* key_backup
		$ ls
		$ nano known_hosts

	Creació d'un parell de claus (aquest pas també el podem obviar si ja en tenim algun de creat; fins i tot podem aprofitar el que ens crea 			l'SparkleShare):
	$ ssh-keygen -t rsa -C "sisco.garcia@ubuntu.cat"
	$ ls

	Còpia de les claus al compte de GitHub:
	$ sudo apt-get install xclip
	$ cat ~/.ssh/id_rsa.pub | xclip -sel clip

	Anem al web i a l'apartat de les claus (https://github.com/account) l'enganxem i li donem un nom.

	Ens hi connectem per ssh i hi afegim els fitxers que vulguem compartir:
	$ ssh git@github.com
	$ git add /home/sisco/Documents/seminaristac/ApuntsMigracio.txt 

	Podem comprovar l'estat del nostre compte:
	$ git status -s


2.- Tot seguit configurar-lo:

	git config --global user.name "Your Name"
	git config --global user.email sisco.garcia@ubuntu.cat
        

3.- Els passos següents són:

	Crear un directori on posar-hi algun fitxer (n'hi ha d'haver un anomenat README) que vulguem pujar (push) al nostre compte de github i crear així 		el primer dipòsit:

	mkdir MigracioPL
	cd MigracioPL
	git init
	touch README
	git add README
	git commit -m 'first commit'
	git remote add origin git@github.com:SiscoGarcia/MigracioPL.git
	git push origin master
      
	Quan ja existeix un dipòsit, el que podem fer és pujar-lo i modificar-lo de la següent manera:

	cd existing_git_repo
	git remote add origin git@github.com:SiscoGarcia/MigracioPL.git
	git push origin master
      
	Després podem anar al directori remot i veure què hi ha i modificar/configurar el que convingui:

	https://github.com/SiscoGarcia/MigracioPL


4.- Des del costat de l'SparkleShare:

	Descarreguem (https://github.com/downloads/hbons/SparkleShare/sparkleshare-0.2-beta1.tar.gz) i instal·lem el paquet, però prèviament satisfem les 		dependències, ja que està creat amb C, amb:

	$ sudo apt-get install gtk-sharp2 monodevelop mono-devel libndesk-dbus1.0-cil-dev libndesk-dbus-glib1.0-cil-dev python-nautilus git-core intltool 		gvfs gvfs-bin python-gtk2-dev openssh-client

	Hem de tenir prèviament instal·lat el make, si no és el cas l'instal·lem fàcilment amb:
	$ sudo apt-get install make

	Descomprimim l'SparkleShare, ens situem al directori descomprimit i ja podem compilar-lo:
	$ tar -zxvf sparkleshare-0.2-beta1.tar.gz
	$ cd sparkleshare-0.2-beta1
	$ ./configure
	$ make
	$ sudo make install
	

	Se'ns haurà creat un parell de claus que s'hauran desat al subdirectori .config/sparkleshare/ de la carpeta arrel (/).
	Hem de copiar el contingut de la clau pública, que serà semblant a sparkleshare.yourname@server.com.key.pub (però amb el nostre correu electrònic 		enlloc de «yourname@server.com.key» al mig del nom de la clau) a l'apartat de les claus (https://github.com/account) del web; l'enganxem i li donem 		un nom (com abans, de fet el pas anterior pot obviar-se si només volem fer servir el compte de GitHub amb l'SparkleShare). Hem de prémer el botó 		«Add key» i ja podem sincronitzar l'SparkleShare amb el GitHub:

	Sincronitzem l'SparkleShare amb aquesta carpeta del GitHub:

	Premem a la icona de notificació de l'SparkleShare i seleccionem «sincronitzar amb la carpeta remota».
	Seleccionem el GitHub com al nostre servidor; entrem el nostre nom d'usuari i el nom de la carpeta a l'entrada «Nom de la Carpeta»; en el nostre 		cas és «Usuari/Carpeta» concretament «SiscoGarcia/MigracioPL».

	Només cal prémer Sincronitza per començar a sincronitzar!

4.2. Per afegir més col·laboradors al Github?

El col·laborador haurà d'aconseguir un compte i afegir la seua clau pública com s'ha descrit abans, però es pot obviar la creació del dipòsit. Una vegada fet, fàcilment es pot afegir la persona a través de la interfície web de Github.

En Github, cal anar al «Dashboard» i seleccionar el dipòsit desitjat a l'apartat «Your Repositories». Llavors cal prémer «Admin», després «Col·laboradors». Cal buscar el nom d'usuari del col·laborador per escriure'l a interior de la caixeta d'entrada de text i prémer «Afegeix» per afegir la persona.

Ja està fet. Ara cada usuari que forma part de l'equip ja pot sincronitzar el seu ordinador amb la carpeta.

5. Compartir fitxers

5.1. Com es poden afegir fitxers?

Només cal posar els fitxers en una de les carpetes del projecte SparkleShare. Pot fer-se això arrossegant fitxers a sobre o desant-los directament des d'una aplicació. L'SparkleShare notarà els canvis. No cal cap més feina manual de sincronització!

5.2. Com compartir els fitxers amb altres?

No només és tot en l'SparkleShare sincronitzat al nostre servidor, l'SparkleShare també ho fa automàticament amb tots altres clients que es van connectar amb ell. Una altra vegada, cap feina extra!

5.3. En cas d'equivocació, com revertir els canvis?

Fent clic amb el botó dret l'arxiu afectat. Hi haurà una entrada del menú anomenada «Aconseguir Versió Anterior». Des d'allà es pot seleccionar una versió anterior de l'arxiu que es vol comprovar. L'SparkleShare crearà una còpia d'aquest fitxer en la mateixa carpeta.
